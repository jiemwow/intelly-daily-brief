# Agent Team Handbook

这份文档定义新项目启动前的默认子 agent 团队、协作顺序和技能基线。

## 目标

建立一套可长期复用的 `5 人固定团队`：

1. `主负责人 / Orchestrator`
2. `产品研究 Agent`
3. `资料验证 Agent`
4. `视觉体验 Agent`
5. `工程架构 Agent`

这套团队优先服务于项目的 `产品定义 + 研究` 阶段，不默认进入编码实现。

## 固定职责

### 主负责人 / Orchestrator

- 负责读取项目背景、拆分问题、派发任务、整合结论、保持方案一致性。
- 输出物：
  - 项目启动说明
  - 任务拆分
  - PRD 草案汇总
  - 最终实施建议

### 产品研究 Agent

- 负责目标用户、核心场景、功能边界、竞品分析、信息架构、优先级。
- 输出物：
  - 项目定义 / PRD
  - 用户场景
  - 成功标准
  - 竞品与差异化结论

### 资料验证 Agent

- 负责官方文档、一手资料、规范、政策、API 事实核验。
- 输出物：
  - 来源可靠的事实结论
  - 官方链接与引用
  - 约束清单
  - 术语边界

### 视觉体验 Agent

- 负责设计参考、版式方向、视觉语言、页面结构、原型建议、设计系统草图。
- 输出物：
  - 视觉方向结论
  - 页面结构建议
  - 风格参考
  - 设计策略

### 工程架构 Agent

- 负责项目骨架建议、技术选型、目录规划、数据流、接口、部署与实施拆解。
- 输出物：
  - 技术栈建议
  - 系统结构草图
  - 模块边界
  - 实施路线

## 默认协作顺序

1. 主负责人读取背景，产出第一版问题拆分。
2. 并行拉起：
   - 产品研究 Agent
   - 资料验证 Agent
   - 视觉体验 Agent
3. 主负责人汇总为第一版 `PRD`。
4. 工程架构 Agent 基于 PRD 输出实施方案。
5. 进入执行阶段后，再按需要补充临时 agent：
   - 数据 / 爬虫 agent
   - 测试 / QA agent
   - 内容编辑 agent

## 冲突处理规则

- 多 agent 结论冲突时，以 `主负责人` 的统一版本为准。
- 涉及事实判断时，`资料验证 Agent` 的结果优先于推测。
- 涉及范围取舍时，优先服从 `PRD`，不提前扩张需求。
- 涉及设计方向时，视觉方案必须同时说明实现成本和约束。

## 技能基线

### 已有技能

- `openai-docs`
- `imagegen`
- `agent-tools`
- `skill-installer`
- `plugin-creator`
- `skill-creator`

### 新装的通用技能

- `doc`
  用于沉淀项目文档、规格和结构化说明。
- `frontend-skill`
  用于网页产品、前端结构、UI 与交互实现建议。
- `playwright`
  用于页面验证、自动化检查和网页任务辅助。
- `screenshot`
  用于竞品截图、页面捕获和视觉对比材料整理。

### 使用原则

- 新项目前期以研究、设计、原型为主，不先堆执行型技能。
- 需要额外能力时，优先由 `skill-installer` 补充，而不是一次性预装很多。
- 官方资料优先由 `openai-docs` 处理；通用搜索与跨模型任务优先由 `agent-tools` 处理。

## 默认沉淀文档

每个新项目至少要沉淀以下 4 份文档：

- `项目定义 / PRD`
- `研究结论 / 竞品与事实`
- `视觉方向 / 设计策略`
- `工程方案 / 实施路线`

模板见 `docs/templates/`。
