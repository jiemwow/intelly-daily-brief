# 架构说明

## 每日流程

1. 定时任务在 `Asia/Shanghai` 时区每天 `08:00` 触发简报生成。
2. 任务从多组 Google News RSS 查询与补充信源中抓取过去 24 小时候选新闻。
3. 候选稿统一整理为共享的 `NewsItem` 结构，并完成来源名称归一。
4. 排序阶段过滤弱相关内容、去重相似报道，并限制单一来源在同一板块内过度占比。
5. 总结阶段输出中文标题、摘要与“为什么重要”的编辑判断。
6. 渲染阶段生成报刊式网页预览和适合邮件投递的 HTML 成品。
7. 当环境变量就绪时，投递阶段通过 Resend 发送邮件。

## 核心模块

- `src/config`
  板块、发送时间与收件人等静态配置。
- `src/types`
  共享领域模型。
- `src/jobs`
  顶层编排入口，包括 `brief:run` 和 `brief:inspect`。
- `src/lib`
  日期、格式化与服务适配等通用工具。
- `src/collectors`
  实时新闻抓取与来源归一。
- `src/rankers`
  板块级筛选与来源多样性控制。
- `src/summarizers`
  中文成稿与重要性判断。
- `src/delivery`
  基于 Resend 的邮件投递。

## 板块模型

- 全球人工智能
- 中国人工智能
- 全球智能驾驶
- 中国智能驾驶

MVP 目标是每个板块优先产出 3 条重点稿件；当高质量候选不足时，允许缩容而不是硬凑。

## 质量规则

- 优先采用官方与一手来源。
- 对评论稿、转载稿和泛财经噪声源降权。
- 对多源交叉确认的事件适度加权。
- 摘要必须简洁，并服务于决策阅读。
- 标题与摘要要清除来源尾巴、站点残留和抓取噪声。
- 部分信源超时或失效时，系统需优雅降级。

## 部署模型

- 部署在 Vercel。
- 用 Vercel Cron 做调度。
- 用 Resend 发邮件。
- 密钥放在 Vercel 环境变量中。

## 设计方向

- 首页参考编辑部首页的阅读路径：头版主稿、侧栏快读、四栏专题。
- 页面优先服务“快速浏览”，不是展示型 landing page。
- 一屏内先回答“今天最重要的变化是什么”，再展开主题分区深读。

## 当前缺口

- 全球智能驾驶板块的高质量英文信源仍需继续增强。
- 当前没有把 Google News 跳转链接解析成文章来源原始链接。
- OpenAI 总结层尚未在真实 API key 环境下完成联调。
