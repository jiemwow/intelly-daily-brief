# 项目定义 / PRD v0.1

## 1. 一句话定义

一个面向个人用户的科技资讯汇聚与每日打卡网站，用结构化简报、分栏阅读和多渠道推送，帮助用户每天高效获取最重要的信息。

## 2. 产品定位

这不是一个无限下拉的综合新闻站，也不是一个复杂的信息社区。  
它首先是一个 `个人科技情报首页`：每天给用户一份已经筛选、排序和结构化过的简报，并通过打卡机制把“看简报”变成稳定习惯。

## 3. 目标用户

- 需要持续追踪科技与商业变化，但没有时间自己筛选信源的个人用户
- 对 `AI`、`智能驾驶`、`具身智能`、`全球要闻`、`商业趋势` 有稳定关注的人
- 希望形成“每天固定看一遍简报”习惯的用户
- 典型用户包括：科技从业者、产品经理、运营、创业者、投资研究兴趣用户

## 4. 用户要完成什么事

- 每天快速知道今天最值得关注的几件事
- 在一个页面里按板块扫读，不必自己四处搜集信息
- 在碎片时间完成一次有效阅读
- 通过打卡建立稳定的阅读节奏
- 在 Web、手机端和微信等 IM 推送之间无缝接续阅读

## 5. 为什么现在值得做

- 资讯越来越多，用户真正稀缺的是筛选、排序和压缩表达，而不是更多链接
- 科技与商业主题日更频繁，用户已经天然具备“每天看一次”的动机
- IM 主动推送降低了回访门槛，打卡机制可以把偶发阅读转成持续行为
- 高质量、结构化、跨端一致的科技简报产品在个人用户场景里仍有明显空位

## 6. 成功标准

### 产品侧

- 用户能在 `3-5 分钟` 内完成一次有效阅读
- 首页能在 `5 秒` 内让用户知道“今天先看什么”
- 打卡与真实阅读行为绑定，而不是纯签到
- Web、手机端和 IM 推送的内容层级一致

### 内容侧

- 每日简报能够稳定生成，不依赖单一来源
- 五大板块每天都能得到基本覆盖
- 头条与重点内容有明显优先级，不是简单堆砌

### 留存侧

- 用户能清楚感知连续阅读和连续打卡状态
- IM 推送能稳定把用户带回站内，而不是停留在消息层

## 7. 核心板块

### 一级核心板块

- `AI`
- `智能驾驶`
- `具身智能`

### 二级环境板块

- `全球要闻`
- `商业趋势`

规则：

- 首页的头条和重点优先从一级核心板块中产生
- 二级环境板块承担背景判断和外部变量补充作用
- 每板块在 MVP 阶段默认展示 `3-5` 条

## 8. 核心场景

### 场景 1：早晨快速扫读

用户在早晨通勤或开始工作前打开首页，先看头条和今日重点，再决定是否深读某个板块。

### 场景 2：收到推送后回站阅读

用户收到微信 / IM / 邮件中的简报摘要后，点击进入站内继续看当天完整内容。

### 场景 3：碎片时间完成打卡

用户在午间或晚间快速看完一轮重点，并完成当日打卡，延续连续天数。

### 场景 4：回看历史简报

用户回看昨天或更早的简报，补读错过的重要内容。

## 9. MVP 范围

- 支持 Web / H5 登录
- 支持每日生成一份总简报
- 支持五大板块的分栏展示
- 首页采用 `头条主稿 + 今日重点 + 分栏快读`
- 支持当日打卡和连续天数记录
- 支持邮件与微信 / IM 作为主动推送渠道
- 支持用户基础偏好设置：
  - 感兴趣板块
  - 推送方式
  - 推送时间

## 10. 打卡机制定义

### 打卡目标

打卡的作用不是单独制造“签到数字”，而是强化“每天读一遍简报”的行为闭环。

### MVP 打卡规则

- 用户打开当日简报页
- 用户点击一次“今日打卡”
- 每个用户每天最多一次有效打卡
- 打卡后更新：
  - `今日已打卡`
  - `当前连续天数`
  - `历史最佳连续天数`

### 后续可扩展规则

- 要求先浏览至少一个核心板块后才能打卡
- 打卡奖励与阅读深度挂钩
- 周报 / 连续阅读奖励

## 11. 首页与阅读结构

首页的阅读顺序固定为：

1. `今日判断`
2. `头条主稿`
3. `今日重点`
4. `五大板块分栏快读`
5. `历史与继续阅读`

原则：

- 先给判断，再给事实，再给扩展
- 用户不需要滚很多屏才能知道重点
- 同一条新闻不在多个位置重复出现

## 12. 多端与触达策略

### Web / H5

- 承担主要阅读体验
- 支持完整板块浏览与历史回看

### 邮件 / 微信 / IM

- 承担提醒与回流入口
- 只给摘要，不承担完整阅读体验
- 所有渠道最终都应回流到站内简报页面

## 13. 明确不做

- 原生 iOS / Android App
- 复杂个性化推荐流
- 评论、社区、关注关系
- 多人协作型后台
- 大而全 CMS
- 实时聊天
- 用户自定义复杂推送编排

## 14. 风险与未知项

- 高质量信源的稳定性和抓取复杂度
- 微信 / IM 渠道的接入方式与送达稳定性
- 打卡规则如何避免形式化
- `具身智能` 板块的内容密度可能波动较大
- 头条与重点的自动排序规则仍需持续调优
