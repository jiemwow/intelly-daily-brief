# 信息架构 v0.1

## 1. 信息架构目标

- 让用户在 `5 秒内` 理解今天最值得看的内容
- 让“读简报”和“完成打卡”形成单条明确路径
- 让 Web、移动端和推送内容共享同一套信息层级

## 2. 核心页面

### 首页 / 今日简报页

用途：

- 承担每天的主入口
- 展示头条、重点、分栏快读、打卡入口

### 历史简报页

用途：

- 查看过去日期的简报
- 作为回看与补读入口

### 板块详情页

用途：

- 查看某个板块当天或历史的内容集合
- 作为深读入口

### 用户页 / 设置页

用途：

- 展示打卡状态、连续天数、推送设置、偏好板块

### 管理页

用途：

- 查看生成状态
- 查看信源状态
- 触发补发或重建

## 3. 首页结构

### 顶部导航区

内容：

- 产品名 / Logo
- 今天日期
- 板块导航
- 登录状态
- 个人入口

### 首屏摘要区

内容：

- `今日一句判断`
- `今日打卡状态`
- `连续天数`
- `头条主稿`

目的：

- 让用户在第一页就得到最重要的信息和行动入口

### 今日重点区

内容：

- `2-4` 条重点新闻
- 以短摘要形式展示
- 点击进入详情或原文

目的：

- 让不想深读的用户也能快速扫一遍核心变化

### 分栏快读区

包含 5 个板块：

- AI
- 智能驾驶
- 具身智能
- 全球要闻
- 商业趋势

每个板块展示：

- 板块名
- `3-5` 条内容
- 每条包含：
  - 标题
  - 简短摘要
  - 来源
  - 发布时间
  - 小图（有可靠图片时）

### 继续阅读 / 历史入口

内容：

- 昨日简报
- 最近 7 天
- 收藏 / 稍后读（后续可扩）

## 4. 首页优先级规则

### 最高优先级

- 头条主稿
- 今日判断
- 今日打卡入口

### 第二优先级

- 今日重点
- 一级核心板块

### 第三优先级

- 二级环境板块
- 历史与继续阅读

## 5. 移动端结构

移动端不做桌面多栏硬压缩，而是改为单列顺序：

1. 顶部导航
2. 今日判断
3. 头条主稿
4. 打卡入口
5. 今日重点
6. 五大板块折叠式内容
7. 历史入口

规则：

- 保持与桌面端相同的信息优先级
- 不因为屏幕变小而改变内容判断逻辑

## 6. 推送内容结构

### 微信 / IM

只包含：

- 标题
- 今日一句判断
- `2-3` 条重点
- 回站链接

### 邮件

包含：

- 今日一句判断
- 头条主稿
- 今日重点
- 五大板块摘要

规则：

- 推送内容是“入口”，不是完整替代站内阅读

## 7. 导航结构

### 一级导航

- 今日简报
- 板块
- 历史
- 我的

### 二级导航

- AI
- 智能驾驶
- 具身智能
- 全球要闻
- 商业趋势

## 8. 用户路径

### 路径 1：主动访问

首页 -> 扫读头条与重点 -> 浏览板块 -> 打卡 -> 离开

### 路径 2：被推送召回

微信 / 邮件 -> 打开今日简报 -> 扫读重点 -> 打卡 -> 浏览板块

### 路径 3：历史补读

历史页 -> 某天简报 -> 查看重点 -> 深读某板块

## 9. 关键组件

- 顶部导航
- 今日判断卡片
- 打卡状态卡片
- 头条主稿模块
- 今日重点列表
- 板块分栏模块
- 新闻条目卡片
- 历史入口模块

## 10. 第一版不进入信息架构的内容

- 评论区
- 社交关注
- 私信
- 排行榜
- 用户生成内容
- 复杂搜索结果页
