# 低保真结构稿 v0.1

这份文档的目标不是定义视觉风格，而是明确：

- 每个关键页面由哪些区块组成
- 用户进入页面后先看到什么
- 打卡、阅读、回看之间的路径如何串起来

## 1. 低保真原则

- 先确保阅读效率，再考虑装饰表达
- 先体现优先级，再讨论视觉层级
- 无图时保持干净文本排版，不做假占位图
- 首页要让用户在 `5 秒内` 明白“今天该看什么”

## 2. 优先绘制页面

第一轮低保真只聚焦 3 个页面：

1. 首页桌面端
2. 首页移动端
3. 板块详情页

原因：

- 首页决定整体产品信息架构
- 移动端决定日常高频使用体验
- 板块详情页决定内容是否真的能被用户读进去

---

## 3. 首页桌面端

### 区块顺序

1. 顶部全局栏
2. 首屏总览区
3. 今日重点区
4. 五大板块分栏区
5. 快速浏览与历史入口
6. 底部功能区

### 结构说明

#### 顶部全局栏

- 左：产品名 / Logo
- 中：全局搜索
- 右：日期、消息、打卡状态、个人入口

#### 首屏总览区

- 左侧主栏：头条主稿
- 右侧辅助栏：
  - 今日一句判断
  - 打卡状态
  - 连续天数
  - 未读数
  - 快捷操作

#### 今日重点区

- 展示 `2-4` 条重点新闻
- 信息密度高于板块区
- 每条只给短摘要和来源信息

#### 五大板块分栏区

- AI
- 智能驾驶
- 具身智能
- 全球要闻
- 商业趋势

每个板块展示：

- 板块标题
- 一句话判断
- `3-5` 条内容
- 每条内容包含：
  - 标题
  - 摘要
  - 来源
  - 时间
  - 小图（仅在图片可靠时显示）

#### 快速浏览与历史入口

- 今日高频关键词
- 最近 7 天入口
- 收藏 / 稍后读入口
- 筛选或跳转入口

#### 底部功能区

- 推送设置
- 反馈入口
- 历史总览

### 线框示意

```txt
+----------------------------------------------------------------------------------+
| Logo        Search                                Date   Check-in   Avatar       |
+----------------------------------------------------------------------------------+
| Lead Story / Headline                         | 今日判断                         |
| Title                                         | 打卡状态                         |
| Summary                                       | 连续天数                         |
| Read More                                     | 未读数 / 快捷入口                |
+----------------------------------------------------------------------------------+
| 今日重点 01 | 今日重点 02 | 今日重点 03 | 今日重点 04                         |
+----------------------------------------------------------------------------------+
| AI                  | 智能驾驶            | 具身智能                            |
| 3-5 items           | 3-5 items           | 3-5 items                           |
+----------------------------------------------------------------------------------+
| 全球要闻            | 商业趋势            | 历史 / 收藏 / 热词                  |
| 3-5 items           | 3-5 items           | shortcut modules                    |
+----------------------------------------------------------------------------------+
```

## 4. 首页移动端

### 区块顺序

1. 顶部紧凑导航
2. 今日主卡
3. 今日重点
4. 五大板块折叠列表
5. 历史与快捷入口
6. 底部导航

### 结构说明

#### 顶部紧凑导航

- 左：品牌
- 中：日期
- 右：搜索 / 消息

#### 今日主卡

- 今日一句判断
- 头条主稿标题
- 打卡按钮
- 今日已读 / 连续天数

#### 今日重点

- `2-3` 条
- 单列排列
- 保持短摘要

#### 五大板块折叠列表

- 默认每个板块显示 `2` 条
- 点击展开查看更多
- 每个板块保留进入详情页的入口

#### 历史与快捷入口

- 最近 7 天
- 收藏
- 稍后读
- 推送设置

#### 底部导航

- 首页
- 历史
- 打卡
- 订阅
- 我的

## 5. 板块详情页

### 区块顺序

1. 板块头部
2. 板块概览区
3. 内容主列表
4. 相关内容与历史入口
5. 底部切换入口

### 结构说明

#### 板块头部

- 板块名
- 板块一句判断
- 更新时间
- 返回首页

#### 板块概览区

- 今日最重要 1 条
- 次重点 `2-3` 条
- 关键标签或趋势词

#### 内容主列表

- 按重要度排序
- 每条卡片包含：
  - 标题
  - 摘要
  - 来源
  - 时间
  - 原文入口
  - 小图（有可靠图片时）

#### 相关内容与历史入口

- 相关主题
- 同板块历史内容
- 收藏 / 分享 / 标记已读

#### 底部切换入口

- 上一个板块
- 下一个板块
- 返回首页

### 线框示意

```txt
+----------------------------------------------------------------------------------+
| < 返回首页    AI / 一句话判断                            更新时间                 |
+----------------------------------------------------------------------------------+
| 今日板块头条                                                                     |
| Title                                                                            |
| Summary                                                                          |
+----------------------------------------------------------------------------------+
| 次重点 01                   | 次重点 02                   | 次重点 03             |
+----------------------------------------------------------------------------------+
| 内容卡片列表                                                                       |
| - 标题 / 摘要 / 来源 / 时间 / 原文                                               |
| - 标题 / 摘要 / 来源 / 时间 / 原文                                               |
| - 标题 / 摘要 / 来源 / 时间 / 原文                                               |
+----------------------------------------------------------------------------------+
| 相关主题 | 历史内容 | 收藏 | 分享                                                |
+----------------------------------------------------------------------------------+
```

## 6. 历史页

### 区块顺序

1. 顶部筛选栏
2. 历史总览列表
3. 展开后的板块摘要
4. 辅助统计区

### 结构说明

- 支持按日期、板块、已读状态筛选
- 按天展示简报摘要
- 每天可展开查看五大板块的简要内容
- 提供打卡和阅读统计概览

---

## 7. 我的页面

### 区块顺序

1. 个人信息区
2. 核心数据区
3. 订阅与偏好
4. 内容管理
5. 账户设置

### 结构说明

- 显示用户当前打卡状态、连续天数、推送设置
- 管理板块偏好、推送方式和阅读记录

---

## 8. 组件清单

这轮低保真至少包含以下组件：

- 顶部导航
- 今日判断卡片
- 打卡状态卡片
- 头条主稿模块
- 今日重点列表
- 板块分栏模块
- 新闻条目卡片
- 历史入口模块
- 底部导航（移动端）

## 9. 低保真验收标准

- 用户能快速区分“头条 / 重点 / 板块快读”
- 打卡入口不被内容淹没，但也不喧宾夺主
- 移动端不是桌面端的机械压缩，而是重新排序后的单列结构
- 板块详情页足够支撑深读，不只是列表页翻版
- 无图状态下页面结构仍然成立
