# 新项目启动流程

## 默认启动顺序

1. `主负责人`
   读取项目背景，输出第一版问题拆分。

2. 并行启动
   - `产品研究 Agent`
   - `资料验证 Agent`
   - `视觉体验 Agent`（如果项目需要较强界面或品牌表达）

3. `主负责人`
   汇总为第一版 PRD 草案。

4. `工程架构 Agent`
   基于 PRD 产出工程方案、技术栈和目录结构建议。

5. 如有需要再补临时角色
   - 数据 / 爬虫 Agent
   - 测试 / QA Agent
   - 内容编辑 Agent

## 推荐工作方式

- 产品问题先澄清，再谈页面和技术
- 一手资料和竞品案例并行推进，不互相替代
- 视觉方向先出原则，再出页面
- 工程方案只在 PRD 基本稳定后输出

## 汇总要求

所有子 agent 的结论都需要被压缩成：

- 关键结论
- 支撑依据
- 风险或边界
- 需要主负责人做的决策
